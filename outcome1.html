1.6 Describing access issues that need taking into account - describe issues that people with disabilities may face when using webpages and how we can overcome them

One way we could help people with disabilities to access web pages is the use of alt tags for the visually impaired. Alt tags just explain what the image is in text so, with software that can read out words to the user the software can pick up and read out what the alt tags say in turn hopefully giving some content on what that image is. Another way we can help people with disabilities is to give subtitles and transcripts when using a video on your web page. This is for people who can not hear very well, would benefit from knowing what's going on without trying to decipher every word that is being said with said video. Having clear navigation like clear to see buttons and links and the right use of colours (smart colours) not only improved on helping the disabled but just anyone who would use your website as the easier people find it as a whole to navigate your website the more likely people who are disabled can start to understand what's going on as well.


1.7 Describe which are the appropriate filetypes for websites

The main 4 that are appropriate for a website .jpeg, .png, .gif and the .svg file type Each of them brings different advantages and disadvantages to them. Like .jpeg being a small file size that is easy to load that displays images. .png has a more high-quality image with more colours and control but is larger and may even not work with older browsers. .gif a small file type that can play a short “gif” that is a short animation within the website and finally .svg that is a vector meaning it's not pixel-based but is made using mathematical formulas making it seem that the quality does not fall in quality by the size or resolution of the image-making it great on different viewing platforms that if the platform allows it.[5]
